# Турнирная сетка — кто угадает (single-file, бесплатный хостинг)

Это **один файл** `index.html`. Никакого сервера. Откройте его локально или
залейте на любой бесплатный статический хостинг. Общие результаты для всех
участников хранятся в бесплатном «бине» на JSONBin.io.

## Как это работает
- Прогнозы семи участников (Митя, Мила, Даша, Алёна, Володя, Даня, Денис) вшиты в файл.
- Сетка **связная**: отмечаете реального победителя матча — он автоматически проходит
  в следующий раунд (там, где была стадия 1/16, теперь видна реальная команда).
  В выпадающем списке следующего раунда доступны только те, кто реально прошёл.
- Таблица очков пересчитывается сама, угаданные ставки подсвечиваются.
- **Без настройки** файл работает в локальном режиме (результаты только
  в вашем браузере). Точка-индикатор слева от кнопки серая.
- **С синхронизацией** результаты общие: их видят все, кто открыл ссылку.
  Точка становится зелёной.

## Общие результаты (через GitHub Gist)
Хранилище уже настроено — это секретный **Gist** на GitHub, его ID вшит в файл.
Отдельный аккаунт-сервис (JSONBin и т.п.) не нужен.

- **Чтение — без настройки.** При открытии страницы приложение само подтягивает
  общие результаты из Gist через GitHub API (свежие данные, лимит 60 запросов/час
  на IP — хватает, т.к. опрос только при загрузке). Зрителям достаточно открыть ссылку.
- **Запись — по GitHub-токену.** Чтобы внести результат:
  1. Создайте токен: <https://github.com/settings/tokens/new?scopes=gist> (classic,
     scope `gist`) → **Generate token**. Или fine-grained с правом `Gists: Read and write`.
  2. Выберите победителя матча (он сам пройдёт дальше по сетке).
  3. Вставьте токен в поле **«GitHub-токен»** и нажмите **Сохранить**.
- **Ссылка для всех** — обычный адрес страницы (кнопка «Скопировать ссылку для всех»
  в окне «Как вносить»). Никаких ключей в ссылке нет.

> Безопасность: токен вводится только в момент сохранения, **в файле не хранится**
> и в ссылку не попадает. Его можно отозвать в настройках GitHub в любой момент.
> Совет: заведите отдельный токен только со scope `gist`.

## Хостинг (любой из бесплатных)

**GitHub Pages**
1. Создайте репозиторий, загрузите `index.html`.
2. Settings → Pages → Source: ветка `main`, папка `/root`.
3. Через минуту сайт доступен по `https://ВАШ-ЛОГИН.github.io/РЕПО/`.

**Netlify / Cloudflare Pages**
- Перетащите файл (или папку с ним) в их Drop-зону — получите готовый URL.

**Локально**
- Просто откройте `index.html` двойным кликом. Синхронизация работает и так,
  но «ссылка для всех» будет вида `file://...` — для общего доступа нужен хостинг.

## Поправить прогнозы
Внутри `index.html` найдите `const DATA = {...}`. Каждый элемент `rows` — это
матч (`slot`) и выбор каждого (`preds`). Имена команд в прогнозах и в выбранном
результате должны совпадать буква в букву.
